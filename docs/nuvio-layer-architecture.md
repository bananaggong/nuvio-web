# NUVIO 레이어 구조 정리

작성일: 2026-05-06

## 한 줄 정의

NUVIO는 대외적으로는 여러 마을/단체의 지역 체류 프로그램이 모이는 탐색 플랫폼이고, 대내적으로는 프로그램 운영자가 신청자와 기수 운영을 관리하는 호스트 운영 툴입니다.

## Layer 1: 공개 탐색 플랫폼

외부 사용자가 보는 영역입니다. 한달살러처럼 여러 단체와 마을의 프로그램을 한곳에서 탐색할 수 있어야 합니다.

### 주요 화면

| 화면 | 역할 |
| --- | --- |
| `/` | 여러 마을/단체의 공개 프로그램 리스트 |
| `/programs/[id]` | 프로그램 상세 |
| `/programs/[id]/apply` | 프로그램 신청 |
| `/villages` | 마을 목록 |
| `/villages/[slug]`, `/[villageSlug]` | 마을별 공식 홈 |
| `/[villageSlug]/[programSlug]` | 마을 안의 프로그램 상세 |
| `/reviews` | 프로그램 후기 모음 |

### 공개 홈의 데이터 원칙

- 공개 리스트에는 운영자가 게시했거나 NUVIO가 검수한 프로그램만 보여준다.
- RSS에서 자동 수집한 후보는 공개 홈에 바로 올리지 않는다.
- 외부 공고 후보는 관리자/호스트 검수 큐에서 프로그램 초안으로 전환한다.
- 마을 프로그램은 마을 페이지뿐 아니라 NUVIO 전체 프로그램 리스트에도 함께 노출된다.

## Layer 2: 호스트 운영 툴

프로그램 관리자와 마을 운영자가 보는 영역입니다. 핵심은 신청 이후의 운영 흐름을 관리하는 것입니다.

### 주요 화면

| 화면 | 역할 |
| --- | --- |
| `/host` | 신청자 파이프라인, 메시지, 증빙, 보고 상태 |
| `/host/villages` | 마을 홈페이지 관리 |
| `/host/programs` | 프로그램 초안/게시 관리 |
| `/host/forms` | 신청서 빌더 |
| `/host/messages` | 안내 메시지 자동화 |
| `/host/reports` | 보고서 자동화 |

### 운영자가 확인해야 하는 것

- 특정 유저가 어떤 프로그램에 신청했는지
- 신청자의 이름, 연락처, 이메일, 신청 답변
- 접수, 검토, 합격, 불합격, 참여중, 완료 상태
- 입금/결제 금액
- 서명, 영수증, 후기 제출 여부
- 기수별 공지와 커뮤니티 안내 상태
- 보고서에 필요한 필드 충족 여부

## 보성 마을 기준 구조

보성 마을 페이지는 NUVIO의 기준 케이스입니다. 마을 홈 안에서 프로그램, 후기, 신청 흐름이 이어져야 합니다.

### 보성 마을 페이지에 포함할 내용

- 마을 소개
- 운영 프로그램 목록
- 각 프로그램 신청 CTA
- 프로그램 후기 모음
- 참여자 커뮤니티 안내
- 숙소/집결/공지 안내 기준
- 문의 채널

### 연결된 보성 프로그램

| 프로그램 | 성격 |
| --- | --- |
| 보성 차밭 문화체험 1기 | 무료 문화체험 |
| 보성 차밭 살아보기 2박 3일 | 체류형 살아보기 |
| 보성 블렌딩티 워크숍 | 유료 워크숍 |

### 신청 이후 흐름

```mermaid
flowchart LR
  A["참가자: 보성 마을 페이지 방문"] --> B["프로그램 선택"]
  B --> C["신청서 제출"]
  C --> D["호스트 콘솔에 신청 기록 생성"]
  D --> E["운영자 검토/선정"]
  E --> F["입금 또는 결제 확인"]
  F --> G["기수별 공지/안내"]
  G --> H["참여 완료"]
  H --> I["후기/만족도 조사"]
  I --> J["보고서 자료화"]
```

## 현재 구현 반영

- 보성 마을 seed에 연결 프로그램 ID를 추가했다.
- 보성 프로그램 3개를 공개 프로그램 데이터에 추가했다.
- 마을 페이지에서 연결 프로그램을 보여주고, 각 프로그램 신청 버튼을 제공한다.
- 마을 페이지에 연결 프로그램 후기 섹션을 추가했다.
- 호스트 콘솔 seed 신청자에 보성 프로그램 신청자 예시를 추가했다.
- 공개 프로그램 목록은 검수되지 않은 RSS 후보 대신 DB 게시 프로그램과 curated seed 프로그램을 보여주도록 조정했다.

## 다음 구현 우선순위

1. 프로그램과 마을의 관계를 seed가 아니라 Supabase `programs.village_id`로 강하게 연결한다.
2. 신청자 테이블에서 마을/프로그램/기수별 필터를 제공한다.
3. 마을 페이지 안에서 `/[villageSlug]/[programSlug]/apply` 형태의 신청 경로를 지원한다.
4. 후기 작성 시 프로그램뿐 아니라 마을에도 연결한다.
5. 보성 기준 기수 관리, 입금 확인, 공지방/후기방 안내 상태를 데이터 모델에 추가한다.
